Introduction

The internet is a physical object. It is a sprawling machine spanning the globe, built from glass, copper, and silicon. And like any physical machine, it has mass, momentum, and (surprisingly) memory, even when nothing is being written to a disk.

By treating the global network as a reservoir of light and electricity, we can derive some surprising properties. We ask four simple questions:

  1. How much data is floating in the wires right now?

  2. Does this information weigh anything?

  3. Could you build a computer that stores nothing, but remembers everything?

  4. When will the internet become heavy enough to collapse into a black hole?

A Computer That Stores Nothing, But Remembers Everything

If the internet holds 2 PB of data, could it be used as a computer? Imagine a machine that has no internal storage (RAM/HDD). Instead, it uses the network itself as a delay-line memory.

The Passthrough Machine

Consider a "head" that sits on a fiber optic loop of length L. It reads bits from the input, looks for simple patterns (like a word in a book), modifies them, and writes them to the output. The "tape" is the loop itself. A bit written to the network travels distance L and returns to the input some time Δt later.

Usefully, this is roughly 64 light-seconds, or a loop extending 1/8th of the distance to the Sun.

Conclusion

A single "Passthrough Machine" using the Earth as a delay line accesses a kinetic memory of 25 MB, sufficient to run a lightweight operating system like Windows 95. To run a modern smartphone OS, your fiber optic loop would need to reach well past the Moon, extending a significant fraction of an Astronomical Unit.

When Will the Internet Collapse Into a Black Hole?

Finally, we ask: What is the minimal loop size required for the internet to collapse into a black hole (Kugelblitz)? We assume the mass of the wire itself is negligible (perhaps supported by antigravity), so gravity assumes only the mass of the data. However, the wire has volume, which limits how tightly we can pack it.

The Density Crossover

For a black hole to form, we must pack the fiber into a ball of radius R that lies inside its own Schwarzschild radius (R ≤ Rs). But we cannot compress the fiber to zero size; we are limited by the physical volume of the glass. The most compact configuration is a solid sphere of cladding.

We have a "density race":

  • Data Mass (M): grows linearly with length L.

  • Event Horizon (Rs): grows linearly with Mass, so Rs ∝ L.

  • Wire Ball Radius (Rball): grows with the cube root of volume (L1/3).

At short lengths (like 1 meter), the physical volume of the wire (Rball) is vastly larger than the tiny event horizon of the light (Rs). But since Rs grows faster (L) than the ball radius (L1/3), there is a critical length where the horizon eventually swallows the wire.

Summary

  1. Global Data in Flight: ≈ 2 PB.

  2. Mass of Information: ≈ 2.3 × 10-16 kg (single bacterium).

  3. Kinetic Memory: 25 MB (Earth Loop) to 320 MB (Moon Loop).

  4. Black Hole Limit: A loop 10⁸⁹ meters long.
